也许这是真正意义上的面经

本来早就该动笔写这篇“读后感”一般的面经，结果却一直缺少动笔的决心和理由，趁今天难得的早起教小学生的闲暇，况且pdd下周一还要接受tx变态面试官的面试，我决定把这个拖了有三个星期的坑填上

这次的面经将包含一些我总结出的“Golden Rule”，以及一些计算机，游戏渲染方面需要关注的点，以及一些关于面试的杂谈

Golden Rule

反正看到这篇面经的都是自己人，名字叫得多大也无所谓吧

You are how you read

这个题目借用了沈向阳博士前些天演讲的题目。你的简历不仅是你的面试官对于你的第一印象，同时也是他面试你时所能使用的关于你的唯一资料。可以说，你是那个为自己的面试制作地图的人。冒险者自己制作地图，这样的rpg难道还能输？

当然有可能，比如说一个为自己制作了地狱难度地图的冒险者，或者一个不按套路出牌的AI（面试官）。

话虽这么说，大部分面试官还是按照套路出牌的，因此，制定一张优秀的地图，让你和你的面试官在面试之后有着舒适的面试体验是非常有必要的，也许这就是你通过面试的关键。

总之，简历是至关重要的，想要给面试官留下怎样的印象，如何提前规划你的面试（真正的面试在面试之前就就开始了），这一切都取决于你的简历。

永远不要和盘托出

我也忘了我在哪看到这句话，大概是这样

“在爱情中，坦诚相待是必要的，但是永远不要和盘托出”

在你的简历中也是这样

在写简历的时候你大概会有这样的困扰：

“简历上没什么好填的啊，自己既没项目经历，也没实习经历，在大学里从头混到尾，除了考试啥也没干，简历该怎么写呢？难道写自己兴趣爱好是打游戏吗？我李白超帅的。”

不要慌，我也这样困扰过，而且，要知道，即便你啥经历也没写，你也有被面试的机会，相反，如果你在你的简历里面胡吹海吹，那么你的面试官总可以找到一堆细节来问你，到时候即便你答出来了，面试官也觉得你也只是简历上写的那样，马马虎虎，更何况你很可能答不出来。

永远别在简历上胡吹海吹，把胡吹海吹的机会留到面试的时候。

当然，你好歹得在简历上写点东西，毕竟这是你的面试时的地图，如果你确实有项目经历，那你确实可以写你在项目里干了什么，用了什么方法。如果你真的非常可怜，啥经历也没有，那你可以写写你平时做的大作业，你在应付考试的时候做的有趣的事情，当然，这些要面向你面试的岗位。

简历中的废话除了让你的简历排版变得好看之外一般没什么用，控制你的简历中废话的数量，面试官一般只会挑一些他们感兴趣的来问你，因此，你可以写一些与岗位无关的，显得你牛逼的废话；同时留下一些面试官熟悉的，可以问你的，让你和面试官有话可聊。

认识你自己

伟大的柏拉图在两千多年前提出了这样一句伟大的口号：“认识你自己！”

面试的时候也是这样的，对于面试时面试官提到的问题，以及你的简历的经历，你都最好对他们进行一个评估：我对于这些到底有多少了解？

在我看来，认识有这么几个层次

1.啥也不懂，听都没听说过：比如我对于编程范式，什么工厂模型什么的，一点都不懂，两次被面试官问到。其实这种啥也不懂的对于面试影响不怎么大，对于那些你不怎么懂的问题，你大可以懂装不懂。面试官允许你有不懂的方面，只要你别太过分，让面试官觉得你在捉弄他就行

2.懂个大概，知道个原理，没有实际写过或者做过：比如我对于操作系统，网络，分布式等概念，只停留在上课听说过的程度，回答这种问题是要比较小心，尽量挑自己确定的回答，如果你比较深入了解某一个方面可以说说，但是切记点到为止。

3.知道原理而且写过代码，做过实验：比如我对于光栅化成像，opengl等概念。如果你确实做过一些实验，比如训练个神经网络，跑个什么并行程序，那么都可以归结为这个层次。这些东西可以多说，你平时是怎么做的，而且因为你写过代码，所以回答起来一般都不慌。

4.做过实验并且深入了解其它相关方法：这些可以说是联想，也是你可以胡吹海吹的地方，比如说面试官跟你提到云，雨等介质，你可以vol light rendering，ray marching，delta tracking balabala讲一堆，或者说他提到path tracing，软阴影，然后你可以guiding path tracing，光场表征，stat统计又讲一堆。比如说人工智能，目标检测领域，你大吹特吹你对于attention，multi task，residual connection，数据增强等等概念的理解。这些是让你跟你的面试官消磨时光的利器，当然，可以胡吹海吹依赖于你对这些概念的理解。

All about those tricks

面试的时候有一些东西需要积累，比如算法，数据结构，闲得无聊的时候，请啃啃算法导论吧。面试官有时候会提一点基础问题上稍微变换一下的问题，不要被吓到，讲一个naive的解决方案也算解决方案，没有问题。除此之外，还有一些套路是可以准备的。

虚函数

只要你提c++，面试官就会问这个——这个是固定套路，建议你弄懂大部分细节，包括有没有虚函数的类的大小，什么时候虚函数表被扩充，什么时候虚函数表被覆盖

如果你学过编译原理，你可以从编译原理的角度考虑这个问题：c++语言只规定了虚函数的作用方式，并没有规定实现方法，如果你是编译器的制作者，你会怎样设计这个编译器来完成虚函数的功能？思考这个问题其实蛮有意思的，如果能从这个角度回答，面试官一定对你刮目相看

智能指针

这个也是固定套路，可以看看c++ primer plus，里面的回答比较丰富，足可以让你应付面试官

网易的面试官问了我个有趣的，套路之外的问题：shared_ptr的循环引用，这个问题其实蛮有意思的，其本质是垃圾回收机制的两种实现方法：引用计数与可达性判断，以及引用计数所蕴含的问题。如果你能从gc的两种方法回答这个问题，面试官应该会很开心的。

领域知识

我只能说说rendering方面的，其实不外乎opengl有几个着色器，opengl中的空间变换，phong光照模型，法贴与tbn矩阵，mipmap原理与level确定，抗锯齿等，这些是面试官的考题，也是你发挥的空间，如果你好好看了games101，你不仅可以完美回答这些问题，你还可以在谈phong的时候讲辐射度量学，光栅化与光追的关系，在谈mipmap的时候谈各向异性过滤，谈抗锯齿的时候谈频域，卷积，sampling pattern，这些应该也够hr们喝一壶了。

杂谈与ACK

首先要感谢giria，pdd，Ebifly，椰浆跟我聊天，分享面试经历，现在，giria已经收到offer，Ebifly应该阿里算法稳了，pdd明天tx面试，希望这篇面经可以帮到pdd和椰浆，也祝大家都可以收到offer，面试顺利。

大部分面试的经历都不怎么好，比如giria被无数面试官放了鸽子，还被讨厌的面试官欺负不好还口，pdd过了一面然后又灰了，椰浆面完一个多月了没啥答复，我也有被tx面试官欺负的经历。

大家要是遇到了糟糕的面试经历不要伤心，就像有那种专门搞面试官，不在意offer的面试者一样，面试官的心态也有好有坏的，有些不那么“职业”的面试官总能想办法搞你一手，问你点奇怪题目，刁难刁难你，（“我最喜欢在面试的时候干人”，SuwakoZhai如是说）遇到这种面试官不要慌张，在掌握尺度的情况下好好回答，不要输了气势就行。

就算没有收到offer，暑假也可以好好玩，还可以做自己喜欢做的事情，失之东隅收之桑榆，不见得糟糕啦。
