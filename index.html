Related Work

频率与采样

顺便把今天看的关于频域和时域的内容总结了

所谓的render可以认为是一个对于渲染函数sample的过程，既然所有函数都可以傅里叶展开，那么一个函数就可以分成线性可加的高频和低频部分，渲染函数也不例外。对于成像，我们只能对于屏幕空间平均地产生有限的sample，这势必导致对图像低频区域的过采样（无效采样）和高频部分的欠采样

对于渲染任务，这将主要产生锯齿的效果，学名走样（aliasing），图形学在旷日持久的反走样（anti-aliasing）斗争中，发现了这样一些方法

  • 更多的采样（SSAA）

  • 将高频信息滤波，然后直接采样（MSAA）

  • 歪门邪道（FXAA）

语义分割

现在已经凌晨一点三十分了，我突然间理解了老博士所谓“博士是不可能在凌晨三点前睡觉”这句话的含义了，想到PDD还在努力地为编译原理的作业奋斗那么我无论如何也应该把这个坑填完

先上张图片，毕竟不能假设所有的读者都知道语义分割(图片随便找的，看得懂就行)

可以很清楚地看见，图片由原本的rgb图片转换成了具有实际语义的各种语义块，比如地面，车辆，行人，建筑，天空等，成功的语义分割对于一些应用具有非常重要的意义，比如无人驾驶等

（这几天在看康德，突然觉得语义分割是不是对应康德先验知性的部分，将不同的感觉组合对于一个物体知觉，之类的）

那么，一个神经网络是如何完成语义分割的过程的呢，语义分割的网络结构有很多种，其中，最广为大家所喜爱的应该是U-net

从卷积神经网络开始讲这个玩意并不现实，对于以前没了解过U-net的同学们来说，可以把这个网络看成两个过程，一个过程，提取图像中包含的关键信息（在rendering中，关键信息就可能是一个mesh的位置，法向，光照等等），而在后半部分进行上采样，把预测的结果（图片）拉大，最后形成输出

Key Contribution

那么，PointRend的关键贡献就在于，他们将一般的神经网络的预测过程看做一个渲染的流程，渲染中，我们通过场景的一些关键信息来绘制图片，而对于神经网络来说，我们可以将后半部分看成对于结果的渲染，而素材则是神经网络所提取的特征，在这一过程中，我们用到了上采样

既然出现了上采样，那么就不可避免的将会出现过采样和欠采样的问题，我们往往对于一个图片的低频部分过采样，浪费了大量的计算，而对于边缘一类的高频区域欠采样，造成边缘的模糊特性。

看这张图，Mask R-CNN对于网球运动员手的那部分分割其实就相当失败，出现了过于模糊的特征

那么，我们应该如何解决这一问题呢，答案同样来自于图形学：在每一个上采样的过程中，区别对待图像的高频和低频区域，对于高频区域，我们使用更多的点进行采样，对于低频区域，我们使用更少的点进行采样

看图就明白了，我们为了完成这个biased的采样，我们只需要一个判断该像素是否处于高频区域的判断标准，就可以完成这一个采样过程，我们可以使用一个单独的神经网络对这些采样到的点进行预测，而对于其他点直接进行插值拉大，这样既解决了低频区域计算量浪费的问题，也解决了高频区域预测困难的问题

Rendering在之前已经准备得差不多了，今天写一些Rendering是对之前的补充

基于着色器的渲染管线

这是我那位好心的一面面试官问我的唯一一个跟渲染相关的问题，一般的渲染管线有三个着色器，顶点，几何，片段，在前期的（old-fashioned）opengl渲染管线中，由于显卡的限制，主要的着色是在顶点着色器中完成的，随着显卡功能的日渐增强，着色的过程从顶点转移到了片段（相比于顶点，片段着色器有着更高的着色精度），顶点在通过几何着色器的解释后，通过光栅化，将一系列的片段属性传给片段着色器，片段着色器经过复杂的着色过程后，再进行一系列的测试（深度，模板），最后将图片绘制到屏幕上

采样 反走样 傅里叶展开

这是一般通过DD（下称PDD）曾经被问到的一个题目，为什么会出现走样，我们如何反走样？

渲染的本质其实是对于渲染函数的一个采样，其实就是存在一个函数f(x,y)->(r,g,b)，我们对于离散的(x,y)找到对应的(r,g,b)，然后画到屏幕上。

学习过傅里叶变换和函数空间的我们知道（其实我们早忘了），任何函数可以由周期性的基函数组合得到，就好像矩阵和基向量一样，那么，一个函数就有“高频”和“低频”的说法，渲染函数自然也不例外，因此，一个图像中将会存在高频和低频信息，低频也就是那些变化不大的区域，比如纯色的桌面，而高频也就是那些变化大的区域，比如说桌面的边缘。如果我们对于高频区域采样不足，那么很容易出现锯齿状的条纹，或者闪烁的纹理贴图，Mipmap可以帮助我们处理贴图的问题，而AA则通过重采样帮助我们解决锯齿状的条纹

SM与AO

PDD在跟小老板讨论可微渲染的时候说应该可微AO来进行可微的阴影渲染，其实AO跟可微渲染的目标并不一致

Phong光照模型中，有三种光照的方式，分别是环境，漫反射和反射，环境光的阴影可以用AO来实现，而漫反射和反射则利用SM，如果我们希望通过可微渲染来达到光源的回归，那么AO其实根本做不到这一点

相对的，AO可以帮我们回归一些有趣的几何信息，因为AO的计算是通过local的几何遮挡进行量化的

渲染的几个空间

我觉得这是个很基础的问题，PDD居然没有被问到，可能是因为PDD的面试官真的好心吧

弄清楚渲染的几个空间和对应的变换对于正确地完成渲染是非常重要的，尤其是对于延迟渲染

总得而言，渲染有这些空间

  • model space
  • world space
  • view space
  • clip space
  • NDC space
  • screen space
  • texture space

模型的三角形定义是在model space中定义的，通过一系列自定义的变换（平移，旋转，切变，对称）我们最终到达了world space

view space是将所有的世界空间中的物体转移到以相机为原点，相机朝向为坐标的空间中，尽管我们在world space中就已经可以完成着色，大部分的着色是在view space中进行的

clip space 相机具有成像平面，在进过投影矩阵后，所有的在view space中的物体被投影到成像平面中，opengl会在这一过程中裁剪掉在成像平面外的片段，因此其被称为clip space

NDC space为了方便，clip space中的坐标会除以齐次坐标，这样带来的好处在于所有的(x,y,z)分量将会保持在[-1,1]之间

texture space则是将NDC space中的所有坐标乘以0.5再加上0.5，因为这样可以与纹理坐标的范围[0,1]对应，值得注意的是，深度缓冲，也即z-buffer中的深度值就是在texture space中

screen space，很多时候称为SS（我也喜欢这个名字），是指所有的变换结束后呈现在观众眼中的空间，这个空间一般来说不涉及什么变换，只是为了给算法取名字方便，比如什么SSAA,SSAO

还有一个TBN space，其实是为了正确应用法线贴图而产生的空间，我们需要将法线贴图转移到面所在的切线空间，就需要TBN矩阵

智能指针

这个应该是c++11的新特性，我印象中有三个（两年前看的cpppp），但是现在只记得两个：unique-ptr与shared-ptr，unique-ptr表示独享指针，一旦被使用，该地址不可被其它地址共享，shared-ptr表示共享指针，大家可以一起用，只有当最后一个指针消亡后，指针所指向地址才会消亡

左值与右值引用

这个其实蛮好玩的，大家有空可以去试试看这段代码

class test {
public:
    int operator[](int& x) {
        return 2;
    }
    int operator[](int&& x) {
        return 3;
    }
};

int main(void) {
    int x = 3;
    test t;
    cout << t[x]<<endl;
    cout << t[3]<<endl;        
    return 0;
}

值得注意的是，很多编译器会自动进行右值引用的优化，关于右值引用，这cpppp里面的话来总结最为妥当：”加入右值引用并不是为了让你真正使用它，而是为了让你方便地使用那些利用右值引用这项语言特性的代码库”。

AA

最先是SSAA，然后是MSAA，在前向渲染中，最常用的还是MSAA，毕竟比起SSAA，MSAA并不用实际地增加片段着色器的调用次数，况且SSAA并不能减少颜色缓冲的大小

FXAA主要用于延迟渲染，尽管MSAA也可能可以用于延时渲染，但是其复杂的设置总是令程序员在延时渲染中不愿意使用它，于是，不如直接把AA当做延时渲染的一部分吧，通过梯度检查边缘，然后插值好像也不赖嘛，也不用管那些讨厌的采样，不也挺好吗？

走样与反走样

我真的忘了那一块怎么做的了，印象中是使用各种各样的卷积核，先变换到一个域然后再变回来什么的，主要是从来没用过

Shader工作流程，正向与延迟渲染

其实正向和延迟渲染在原理上并没有什么不同，对于一个三角形来说，正向和延迟渲染其实运算流程是一样的，都是对像素插值，然后确定颜色，延迟渲染的唯一好处在于每个片段只用调用一次片段着色器，在片段着色器花里胡哨的今天，这个一可以统一接口，二可以减少调用，即便是正向渲染，其部分效果，如shadow map，也需要延迟的流程进行后处理

AO/Shadow

AO和Shadow总容易让人弄混，AO其实削弱的是环境光照，在削弱环境光照的过程中，会给人一种shadow的感觉，但其实其并不是真正的shadow。SM与RSM后来还成为one-bounce全局光照的经典算法，今天面试完之后，我应该会专门写一些通过延时渲染完成全局光照的blog

法向/法向贴图/Mipmap

这些其实还好啦，只要记得把法向转到切平面就行了，mipmap本质是一个插值算法，本身其实并不会在复杂度上优化渲染流程，但是通过插值的方法，使得远处的纹理更加真实

人间喜剧

发表于 2020-03-18 | 分类于 People ， HyoukaRin

前言

显然，这个题目是在致敬巴尔扎克，伟大的巴尔扎克在年轻时立志用笔对当时法国的大小故事，风土人情进行全面的考察，内容涵盖王公贵族，城市小民，艺术理论，哲学思考等方方面面，最后成就了人类历史上最好看，最有趣的中短篇小说集。

取这个相同的题目并不是说我立志成为像巴尔扎克一样的小说家，只是因为我仰慕于年轻的巴尔扎克的雄心壮志，觉得自己也有必要为未来的一些事情做好计划。

写下这份record，也算与读者共勉。

Here is my plan

每天早上八点半起床

至少我在大一，心里曾经还是有这样的梦想，最后没有了早八，渐渐被社会磨平了棱角。

我觉得不管怎么样，八点半起床还是蛮重要的，即便是八点半起床看小说看漫画看动漫，也应该早上八点半起来，以便自己可以看更多的小说动漫和漫画。

每天睡觉前看哲学史，社会学或者短篇小说。

但是这三种书确实让我有很爽的经历，尽管这些书不比其它书可以为我带来及时的收益或者娱乐，你总会在不经意的时候想起这些书的只言片语，从而意识到这些书为你带来的长期收益，况且，哲学和社会学说不定还会起到催眠的功效。

即便有再多的论文压着，至少要在早上写代码。

这些年养成了空谈论文吹比不想写代码的糟糕习惯，究其原因，南京大学的CS教育理念可能难辞其咎。

我对自己的代码风格还算满意，但是不应该养成见人吹比，不想动手的风气，况且还有一堆没有完成的project。

  • pytorch based rendering

  • 小鸨的nips和小老板的global illumination

  • blog（这个已经完成了，可喜可贺）

  • Animate Gan

  • 渲染开发或者再跟龙哥打比赛

永远不再星期天工作

让我在星期天工作的甲方见鬼去吧，我要在星期天玩游戏，看动漫，出去吃东西，旅游，看轻小说。

晚上去健身

既然写下来了，我当然要认真对待，生完病后我算是知道，活着真好，得好好活着。

ACK
